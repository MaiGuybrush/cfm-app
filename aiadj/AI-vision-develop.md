目前針對我們部門所負責關於影像的系統跟應用程式進行一個整理，這些應用主要是讓工廠智能化，減少人員負擔。
目的是希望能讓副總得以快速了解我們部門目前為工廠提供的解決方案及未來的發展方向。

[影像系統與應用程式整理]

[INX ADJ(auto defect judge)]
description = 用於接收AOI機台產生的缺陷圖片進行自動缺陷判斷的影像系統，可降低人員覆判的負擔，原本只有透過周期比對找出defect的功能，後來可結合AI ADJ系統，減少人員設定的設定的負擔，提高準確性與效率。
target = 提供AOI機台缺陷圖片自動判斷功能，功能與廠內MES系統及AOI機台應用有深度整合。
status = 已完成

[AI ADJ(auto defect judge)]
description = 提供影像建模功能，包含物件分類、物件檢測、語義分割等功能，並提供API供edge端程式使用，設定讀取目錄取得圖片進行判斷，並將結果回傳到指定目錄。使用者可透過此edge 端程式進行應用開發。
target = 提供影像建模功能，包含物件分類、物件檢測、語義分割等功能，功能與廠內MES系統及AOI機台應用有深度整合。
status = 已完成

[INX ADM(auto dimention measurement)]
description = 可從圖像中選定量測標的進行距離和尺寸的自動量測，減少人員量測的負擔，提高量測的準確性和效率。也可結合AI ADM系統，針於難以設定threshhold的目標。INX ADM也提供glass ID 二維碼辨識功能，確保機台刻號品質。
target = 提供影像量測功能，包含距離量測、尺寸量測及二維碼辨識功能，功能與廠內MES系統及AOI機台應用有深度整合。
status = 已完成

[UMS(Universal Vision System)]
description = 建立一個標準化、集中式、且可擴充的通用影像建模平台，為公司各部門提供影像資料上傳、標註、資料管理、模型訓練、模型管理與部署的一站式服務。
target = 提供通用影像建模系統平台，包含物件分類、物件檢測、語義分割等功能。提供泛用場景的應用及較為彈性的edge 端應用程式，使用者無需開發程式；另外也提供API，可與外部系統整合(如:HOP)或自行開發應用。
status = 2026 Q3規劃上線

[HOP(Human Operation Platform)]
description = 結合UMS, HOP讓使用者針對以下場景進行實時影像監控與分析：1.人員作業流程監控 2. 環安監控 3.設備運轉監控。HOP提供即時影像串流與事件通知功能，讓使用者能夠即時掌握現場狀況，提升工廠的智能化水平。
target = 提供實時影像監控與分析平台，結合UMS進行多場景應用，提升工廠智能化水平。
status = 2026 Q3規劃上線

[專案式AI影像應用開發(AI Vision Inspection AP)]
description = 針對特定工廠需求，進行專案式AI影像應用開發，包含需求分析、系統設計、模型訓練與部署等，提供客製化的解決方案，協助工廠提升生產效率與品質。
