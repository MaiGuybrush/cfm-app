# Title GPU 算力方案評估

## slide 1 GPU 算力需求

### 現況

- 推論: 數值/影像類模型算力需求小，且多數應用於edge 端建置獨立GPU即可。通常沒算力問題。

- 建模: 影像類算力需求較大，目前大多集中在AI ADJ 的Model Server，今年採購新機後，效能提昇且後續MA費用大幅降低。

## slide 2 自建/租用算力費用比較(以AI ADJ為例)

- 算力需求：依照ADJ平台訓練 4800 個 YOLO v4 模型估算，每個模型需 2.5 天（60 小時）V100 GPU。 總算力需求：4800 × 60 = 288,000 V100 GPU 小時。 
- 評估範圍：台灣雲端租用 vs. 自建 Model Server（V100 基準，H100 加速替代）。 
- 假設：並行排程（Kubernetes），無資料傳輸瓶頸；企業級定價（非學術）。

### 自建 Model Server 優缺點分析

- **優點**：
  - **長期成本效益**：初期投資高，但長期（如6年）攤提下來，單位成本相對較低。
  - **資料安全性**：資料完全落地，不需上傳雲端，符合資安最高要求。
  - **使用彈性**：內部排程彈性高，可隨時執行各種實驗性或機密性專案。

- **缺點**：
  - **初期投資大**：需一次性投入大量資本支出。
  - **維運成本**：需專人維護硬體、軟體環境與網路，產生額外人力成本。
  - **技術升級延遲**：硬體規格在採購當下即固定，無法像雲端服務一樣隨時取得最新技術（如下一代 GPU）。

## slide 4 結論與建議

1.  **短期/臨時需求 (Cost-Effective Choice)**：
    - **租用算力** 超出公司內部可提供算力，可考慮租借外部算力，但需考慮機敏資料保護問題。

2.  **長期策略 (Strategic Investment)**：
    - **自建 Model Server** 仍是核心競爭力的保障。建議持續投資，特別是當涉及公司核心演算法與機敏資料時，自建方案的安全性與彈性無可取代。

### 綜合建議

- **混合雲模式**：建議採取「**自建為主，租用為輔**」的混合模式。
  - **日常研發與核心專案**：使用內部 Model Server。
  - **爆發性/臨時性算力需求**：優先考慮租用 GMI Cloud 的 H100 服務，以最低成本快速完成任務。
- **行動方案**：
  - **立即行動**：與 GMI Cloud 及 TWCC 建立聯繫管道，申請試用帳號，驗證其服務穩定性與效能。
  - **持續評估**：定期（如每半年）重新評估市場租用價格與自建成本，動態調整算力策略。
