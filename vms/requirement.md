## 專案概述

### 系統目標
建立一個標準化、集中式、且可擴充的通用影像建模平台，為公司各部門提供影像資料上傳、標註、資料管理、模型訓練、模型管理與部署的一站式服務。

### 業務價值
- 統一建模流程，減少重複工作與錯誤率
- 集中管理計算資源（GPU 叢集），提高資源使用效率
- 建立模型資產庫，便於重現與審計
- 降低各部門導入影像 AI 的技術門檻

### 專案範圍
**包含範圍 (In-scope)**：
- 影像資料上傳與存放
- 標註工具整合
- 訓練工作流程與 GPU 排程
- 模型版本管理與部署
- 權限與審計
- 開放 API 介面
- 提供標準Edge端推論程式

**排除範圍 (Out-of-scope)**：
- 現有 AIADJ/廠務 系統的直接改造
- 非影像型別的模型訓練（除非有明確需求）

## 功能性需求

以下功能需求依照更新後的專案需求整理，涵蓋專案管理、專案內的資料集與模型管理、佈署、使用者/團隊管理、API 管理、GPU 資源管理與 SSO 整合。

### FR-1. 使用者專案管理
- **FR-1.1** 使用者能建立、編輯及刪除專案
- **FR-1.2** 專案能分享給團隊成員並支援權限設定（Owner / Editor / Viewer）
- **FR-1.3** 每個專案可設定預設計算配額、可見資料集與模型範圍
- **FR-1.4** 提供專案層級的活動與審計日誌

### FR-2. 專案內容：資料集管理 / 模型管理 / 佈署管理
#### FR-2.1 資料集管理（Dataset）
- **FR-2.1.1** 支援資料集的建立、維護與刪除
- **FR-2.1.2** 支援下列資料型態：
   - 圖像 + 單一標注
   - 圖像 + 多標注
   - 圖像 + (矩形框 + 標注)
   - 圖像 + (多邊形框 + 標注)
- **FR-2.1.3** 資料集版本管理與樣本瀏覽
- **FR-2.1.4** 支援資料集匯入/匯出與統計資訊

#### FR-2.2 模型管理
- **FR-2.2.1** 選擇模型架構並指定合適資料集進行建模
- **FR-2.2.2** 支援增量訓練（fine-tuning / continued training）選項
- **FR-2.2.3** 支援對既有模型進行重新訓練（retrain），並保留歷史版本及元資料(建立者、時間、版本、超參數、評估指標等)
- **FR-2.2.4** 模型版本資訊須包含：版本標籤、建立者、訓練參數、評估指標、artifact 連結
- **FR-2.2.5** 提供刪除模型功能
- **FR-2.2.6** 提供模型建置 / 訓練狀態查詢 API 與介面

#### FR-2.3 佈署管理
- **FR-2.3.1** 佈署管理包含部署設定、流量管理、回滾與監控整合
- **FR-2.3.2** 提供標準 Edge 端推論程式

### FR-3. 使用者 / 團隊管理
- **FR-3.1** 使用者帳號與團隊管理，團隊可擁有多個專案
- **FR-3.2** 權限模型支援 Owner / Editor / Viewer 與專案層級存取控制
- **FR-3.3** 提供團隊邀請、成員管理與角色指派介面

### FR-4. API 管理
- **FR-4.1** 提供 REST/GRPC API 與 Webhook 支援
- **FR-4.2** API 權限控管、金鑰管理及稽核日誌
- **FR-4.3** API 使用率防護（Rate limiting）與監控
- **FR-4.4** 提供 API 文件與測試介面（如 Swagger / OpenAPI）

### FR-5. GPU 資源管理
- **FR-5.1** 管理 GPU 叢集/節點的資源配額與排程策略
- **FR-5.2** 支援排隊、優先權、資源限制與多租戶隔離
- **FR-5.3** 提供資源使用監控、歷史紀錄與計費/配額報表
- **FR-5.4** 提供查詢當前資源使用狀態的 API/介面

### FR-6. 整合現有 SSO 機制
- **FR-6.1** 與公司 SSO/OIDC/LDAP/AD 整合，支援 SSO 登入與單一來源的使用者授權
- **FR-6.2** 支援角色映射與選項性的自動同步
- **FR-6.3** 記錄身份驗證與授權的審計日誌

### 備註
- 原先文件中的標註管理、監控、審計等詳細項目仍保留於相關章節；若需要，可把 FR-2.1/FR-2.2 等子項再展開為更細的需求條目（例如標註任務、標註版本化、實驗追蹤等）。
